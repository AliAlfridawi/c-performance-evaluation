# Abstract

This project benchmarks quick sort, linear search, and binary search across C, Java, and Python using a shared-input protocol. The earlier version of the repository mixed language-specific random generators, single-run timings, and stale result artifacts. The current version corrects those issues by reading deterministic datasets from disk, running warm-up and measured trials separately, batching tiny workloads to avoid zero-time rows, and summarizing results with medians instead of single samples.

The regenerated data shows that comparison counts match across all three implementations for every published configuration. At `N=5000`, quick sort on random input has median in-process times of about `0.000132 s` in C, `0.000316 s` in Java, and `0.012126 s` in Python under this Windows-based harness. Linear search scales directly with hit position, while binary search remains near logarithmic comparison counts regardless of language. The benchmark therefore supports a narrow claim: algorithmic work can be held constant across languages, while the observed in-process execution cost still differs substantially across runtimes in this setup.
